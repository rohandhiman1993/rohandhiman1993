# Unknown

The unknown type is pretty similar to any type in TypeScript

Now, I won't say that unknown type is super helpful, but it is any day better than the any type because

here TypeScript forces you to check the type of particular value before you use it within your program.

function multi(number: unknown) {
  // getting error warning
  return number * 2;
}

function multiply(number: unknown) {
  // developer can check and can handle it. now there is no error.
  if (typeof number === "number") {
    return number * 2;
  }
  return "Please provide a valid number";
}

# Union type

A union type can either be a dog or it can be a cat, or it can be both cat and dog.

One thing to note here, a union type has to have all the properties of one of the types so it cannot have a missing property.

type Dog = {
  name: string,
  barks: boolean,
  wags: boolean,
};

type Cat = {
  name: string,
  purrs: boolean,
};

// use any objec cat or dog or we can use both properties
type DogAndCatUnion = Dog | Cat;

// only dag use
let dog: DogAndCatUnion = {
  name: "buddy",
  barks: true,
  wags: true,
};

// only cat use
let cat: DogAndCatUnion = {
  name: "bella",
  purrs: true,
};

//both use  combination of cat and dog
let dogAndCat: DogAndCatUnion = {
  name: "hybrid",
  barks: true,
  wags: false,
  purrs: true,
};

# Union types with primitive types

// function addNumbersString(a: number | string, b: number | string) {}

type stringOrNumber = number | string;

function addNumbersString(a: stringOrNumber, b: stringOrNumber) {
  if (typeof a === "number" && typeof b === "number") {
    return a + b;
  } else {
    return a.toString() + b.toString();
  }
}

console.log(addNumbersString(3, 3)); // 6
console.log(addNumbersString("5", "5")); // 55

# Intersection types

So this is the major difference between a union and an intersection.

Union types can be of either type dog or can be a cat type, but in case of intersection, it is all the unique properties combined with the common properties of the two types.

type Cat = {
  name: string,
  purrs: boolean,
  color: string,
};

type Dog = {
  name: string,
  barks: boolean,
  color: string,
};

// Now, TypeScript will take all common properties from the dog type as well as from the cat type first.
type HybridAnimal = Dog & Cat;

let hybridAnimal: HybridAnimal = {
  name: "max",
  color: "white",
  barks: true,
  purrs: true,
};

# Truple

A tuple is a typed array with a pre-defined length and types for each index.

Tuples are great because they allow each element in the array to be a known type of value.

To define a tuple, specify the type of each element in the array:

// let person: [string, string, number];
let person: [string, string, number?]; // optional properties

person = ["rohan", "dhiman", 30];

// lists of students

type listOfStudents = [number, boolean, ...string[]];

const passingStudents: listOfStudents = [
  3,
  true,
  "rohan",
  "charu",
  "nisha",
  "divya",
];

console.log(passingStudents); // [ 3, true, 'rohan', 'charu', 'nisha', 'divya' ]

const failingStudent: listOfStudents = [1, false, "divya"];
